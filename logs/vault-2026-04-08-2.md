## Decisions

- **Package name `thevoidforge`**: `voidforge` taken on npm by yangshun. Methodology package: `thevoidforge-methodology`. Both published.
- **LAN mode = full access**: LAN (ZeroTier/private network) is MORE secure than remote, not less. Removed all endpoint restrictions. Auth enforced via password (no TOTP).
- **Pattern files duplicated**: 9 pattern files exist in docs/patterns/ (methodology) AND packages/voidforge/wizard/lib/patterns/ (runtime). prepack-patterns.sh syncs them. ADR-039 trade-off.
- **Methodology bundled in dist/**: copy-assets.sh copies CLAUDE.md, commands, methods, patterns into dist/ for project creation. 452 files in package.
- **v22.0 architecture (ADR-040)**: Everything project-scoped. URL-based routing `/api/projects/:id/*`. Financial paths move from `~/.voidforge/treasury/` to `project/cultivation/treasury/`. WebSocket scoped per-project. Lobby shows aggregated KPIs.
- **WebSocket origin for LAN**: terminal.ts had its own origin whitelist that rejected ZeroTier IPs. Fixed to match server.ts CORS handler (accept private IPs in LAN mode).

## Failed Approaches

- **tsx shebang in npm package**: `#!/usr/bin/env npx tsx` doesn't work when installed from npm. tsx is a devDependency, not bundled. Fixed: shebang → `#!/usr/bin/env node`, compile to JS via tsc.
- **`voidforge` npm name**: Already taken. Had to rename to `thevoidforge`.
- **npm granular token with passkey**: First token type couldn't publish (OTP required). Needed granular access token with publish permissions to bypass passkey/OTP.
- **GitHub Actions billing**: Account locked on billing, couldn't run publish workflow. Published locally instead.
- **LAN endpoint restriction (v21.0.2→v21.0.3)**: First fix added project endpoints to allowlist. User correctly pointed out LAN should have NO restrictions. Removed entire block.
- **process.cwd() everywhere**: 7+ modules used CWD to find project files. Standalone wizard's CWD is wherever you launched it. Fixed with findProjectRoot() marker discovery + explicit projectDir params.
- **Blueprint detection on CWD**: Wizard scanned CWD for PRD.md, found VoidForge's own template PRD. Fixed to scan target project directory instead.
