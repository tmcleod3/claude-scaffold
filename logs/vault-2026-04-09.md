## Decisions

- **Per-project financial data, not global:** Stark/Riker argued financial data should stay global (ad platform spend is per-account). Overruled — per-project is correct because daemons run per-project and campaigns are project-scoped. Dissent preserved in ROADMAP.
- **Clean break migration:** Per-project logs start fresh (genesis hash). Global data archived, not copied. Copying creates false provenance (Riker/Spock).
- **Router upgrade in M0, not M5:** One URL migration beats two. Riker challenged Picard's query-param workaround and won.
- **Writes before reads (M2↔M3 reorder):** Daemon write paths must move before dashboard read paths. La Forge identified split-brain risk.
- **WebSocket subscription rooms (Option B):** Single WSS with projectId filtering, not per-project WSS instances. Kusanagi design.
- **Backward-compat legacy routes:** Old `/api/danger-room/*` paths preserved as shims for existing UI. Sunset in v23.0.
- **ADR-042 — Dynamic agent dispatch:** Cross-domain spot-checks, content-driven selection, promote 6 high-value agents to frequent use. Fixes static command manifests.
- **ADR-043 — Max by default:** Invert flag taxonomy. Default is autonomous + full roster. `--blitz`/`--muster` retired as no-ops. New opt-out flags: `--light`, `--interactive`, `--solo`.
